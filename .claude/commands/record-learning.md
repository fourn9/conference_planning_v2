# /record-learning

セッション中の発見を `LEARNINGS.md` に保存する。

## いつ呼ぶか

- フェーズ実行中に「これは次回も役立つ」と思える発見があった
- ユーザーの選好・規約・環境について学んだ
- 既存スキルでは捕捉できていなかった文脈を発見した

## いつ呼ばないか（anti-noise）

- 単なる作業ログ（「ファイルを編集した」「テストを走らせた」）
- PR#・Issue#・commit SHA を含む取引的記録
- 推測・仮説（事実として書かない）
- ユーザーへのネガティブな評価

## 実行手順

1. `LEARNINGS.md` を Read で読む
2. 末尾に以下のフォーマットで追記する:

```markdown
## YYYY-MM-DD — <一行サマリ（30 字以内）>

**What**: 何が起きたか・何を発見したか（2-3 行）

**Why it matters**: なぜ重要か・どのフェーズに効くか（1-2 行）

**Source**: セッションの文脈・元となった会話（任意、1 行）

**Tags**: #phase-N #role-X #anti-pattern など（任意、1 行）
```

3. 追記後、ユーザーに「以下を `LEARNINGS.md` に追記しました」と差分を提示
4. **declarative で書いたか確認** — 「X を好む」「Y の傾向」など観察形。命令形（「常に X せよ」）になっていたら書き直す

## 引数

引数なしで呼べる（agent が文脈から内容を決める）。
ユーザーから明示的に内容を指定された場合はそれを優先する。

## 関連

- `CLAUDE.md` §C.2（書き込みトリガ）
- `CLAUDE.md` §C.3（declarative > imperative）
- 同じ学びを 3 回書いていることに気づいたら `/draft-adr` を提案
- 学びがスキル化できそうなら `/draft-skill` を提案
