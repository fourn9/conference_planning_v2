# スキル：リサーチ成果のレビュー

## 目的

`research/design-prompt` で設計したプロンプトを LLM に投げて返ってきた**成果ドキュメントを、使う前に品質レビュー**する。
ソース単位の採否（`research/hardgate-evaluation` の G1-G7）を内包しつつ、ドキュメント全体の信頼性・網羅性を点検する。
問題が出たら `research/iterate-from-failures` で次プロンプトに反映する。

## レビュー観点（6つ）

### 1. 実在性監査（最優先）
- 引用 URL を**実際に開いて**実在・本文一致を確認（捏造・リンク切れ・要約の取り違えを弾く）
- 著者が特定でき、職種/所属が裏取りできるか
- → ソース1件ずつに `research/hardgate-evaluation`（G1-G7）を適用

### 2. カバレッジ
- プロンプトが求めた範囲（全セル・全対象）に対し、**埋まっていない箇所（空セル/欠落領域）を明示**できるか
- 「該当なし」が正直に報告されているか（無理に埋めた痕跡がないか）

### 3. 引用品質
- 主張ごとに**本文引用付き**で根拠が示されているか（M:N の対応：1記事が複数論点を裏付けてよい）
- スニペット止まり・推測で補った箇所がないか

### 4. カテゴリ越境
- 隣接カテゴリ（隣接職種・対象外セグメント）を当該枠に紛れ込ませていないか
- 同一ソースが「枠Aでは可・枠Bでは不可」を正しく仕分けているか

### 5. closed list 偏り
- 特定企業・特定の常連・自社過去イベントに偏っていないか（新規発掘が死んでいないか）

### 6. 他社事例の自社誤用
- 業界トレンド解説・他社事例まとめを、**当事者の現場判断**と取り違えて採用していないか

## 出力

- 各項目を **採用 / 要再検証 / 却下** に仕分け（理由つき）
- **空セル・欠落領域の一覧**（＝次ラウンドで埋める対象、または仮説見直しフラグ）
- 観点1-6 で見つけた**問題の類型**（`failure-patterns.md` の番号に紐づける）

## 判断基準

- 1件でも捏造・実在不能が見つかったら、その成果は**信頼度を落として全体を疑う**（一部捏造は全体汚染の兆候）
- カバレッジ不足は「悪い成果」ではなく「探索範囲の見直し materials」として扱う

## 次の一手
