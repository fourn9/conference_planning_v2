# スキル：ハードゲート評価（G1-G7）

## 目的

リサーチ候補（URL、ソース、または人物）を Matrix B または登壇者ショートリストに
受け入れる前に、7 つの必須チェック（G1-G7）を適用する。

## このスキルを呼び出すとき

- Phase 3 Matrix B 構築時（URL ごと）
- Phase 5 登壇者リサーチ時（人物ごと）
- 候補結果の証拠が弱いとき

## 7 つのゲート

### G1：識別可能性
- ソース：明示的な著者氏名と現所属
- 候補：氏名、現雇用主、役職が明示的に見える
- ❌ 匿名投稿、❌ 帰属なし wiki 記事、❌ 「X 社のエンジニア」

### G2：直近性（2024+）
- アウトプットは **2024-01 以降** の日付（イベント日に応じて調整）
- PMF 前／非常に早期の企業向けには 2021+ も許容（カバレッジ少）
- ❌ 2022 年以前の書籍／登壇（時代を超えていて再引用されていない限り）

### G3：当事者経験
- 著者／候補がトピックの直接的・現在的な運営経験あり
- 彼らの会社が実際に作っている／運営している／実装している
- ❌ 「マイクロサービスパターンのサーベイ」（概念記事、当事者なし）
- ❌ 候補が他者をまとめているインタビュー、本人がやっていない

### G4：検証可能性
- URL が生きていて引用が再現可能
- 名前が引用ソースに登場
- 候補の述べた役職が会社の公式ページに存在
- ❌ 「私的会話で X が私に話したように」（引用なし）

### G5：厳密カテゴリ適合
- Matrix B セル向け：特定の職種に一致しなければならない（BE 行 != セキュリティ行）
- 登壇者テーマ向け：特定のテーマに一致、親領域だけではない
- ❌ セル内の隣接職種（TL セルの VPoE 記事は G5 失敗）

### G6：タイトルを超えた実質
- 候補が実際に考えを述べているアウトプットがある
- タイトル／職務内容／会社プレスリリースだけではない
- ❌ 「X 社のチーフアーキテクト」 — 本人の執筆なし
- ❌ 登壇者バイオ段落（実際の登壇内容なし）

### G7：文脈一致
- コンテンツの全体文脈がテーマに一致
- 偶然出現したキーワードだけではない
- ❌ 記事で「マイクロサービス」が 1 度だけ言及、マイクロサービス専門知識として扱う

## 手順（エージェント向け）

### Step V.1：候補ごと

ソース／候補証拠を読む。各ゲート G1-G7 をチェック。

### Step V.2：ゲートごとマーク

```
G1 ✅/❌  理由
G2 ✅/❌  理由
G3 ✅/❌  理由
G4 ✅/❌  理由
G5 ✅/❌  理由
G6 ✅/❌  理由
G7 ✅/❌  理由
```

### Step V.3：判断

- 7 つすべて ✅ → **採用**
- いずれか ❌ → **却下**（理由をログ）

「候補なし」は許容。リストを水増しするためにゲートを **絶対に** 緩めない。

### Step V.4：ログ

採用候補：引用に G1-G7 チェック状況を含める。
却下候補：`rejected-candidates.md` に失敗理由付きでログ。

これが将来の再リサーチで、すでに却下された候補を再評価することを防ぐ。

## ゲートを緩めてよい場合（とその方法）

ゲートを緩めるのは、明示的な文書化を伴う場合のみ：

- **G2 緩和**（許容可能な古いソース）：なぜこの 2022 ソースが依然として権威あるかを
  文書化（例：安定概念の標準参照）
- **G3 緩和**（背景知識としての概念記事）：「背景知識」としてマーク、セル証拠ではない。
  別の「前提知識」セクションに配置（`methodology/conference-planning-framework.md` §4.6.4）

G1 は絶対に緩めない（匿名ソースは却下のまま）。

## ゲートをスキップした場合の失敗モード

- Matrix B セルが概念記事で埋まる（当事者なし）
- 登壇者ショートリストが「名前認識」で埋まる（トピック一致ではない）
- 再リサーチが既に却下された候補で時間を無駄にする
- 批判的見直し（Phase 4 / 4.6）が弱いテーマを修正には遅すぎる段階で発見

## 隣接職種ルール（G5 具体）

Matrix B 向け：

| 隣接役割 | デフォルト配置 |
|---|---|
| VPoE | EM 行、または独自セクション |
| アーキテクト | TL 行（職務記述が重なる場合のみ） |
| データエンジニア | データ行（2026-05 から Matrix B に追加） |
| ML エンジニア | 「前提知識」セクション（Matrix B 行ではない） |
| TL ハンズオン付き PMF 前 CTO | TL 行 OK |
| IT 大手 / 成長期 CTO | Matrix A のみ（Matrix B 行ではない） |

同じ著者が一方の行で G5 を通過し、別の行では失敗することがある。
**G5 は行ごとにチェック**、著者ごとではない。
