# スキル：Matrix B ジャンル集計

## 目的

過去イベントデータがない場合の「過去セッション分布」の代替。
Matrix B 全セルにわたる「知りたいこと」をジャンル別に集計し、分布代理を生成する。

## このスキルを呼び出すとき

- 過去イベントデータなしで Phase 4.6 に接近
- `conference-planning.yml` に `use_historical_data: false` または `is_first_time: true`
- テーマバランスの分布チェックが必要

## 手順（エージェント向け）

### Step A.1：この領域のジャンルを定義

デフォルトジャンル（適応）：

- AI 関連（AI が主レンズになる任意のトピック）
- アーキテクチャ刷新（モダナイゼーション、レガシー）
- マルチプロダクト / マルチサービス
- データ基盤
- 信頼性 / 可観測性
- セキュリティ
- インフラ / プラットフォーム
- 組織 / プロセス

### Step A.2：各「知りたいこと」エントリにタグ

Matrix B の 42 セルそれぞれについて、「知りたいこと」の各番号付きエントリに
1 つ以上のジャンルでタグを付ける。

（エントリは複数のジャンルタグを持てる。例：「Agent のための認証付き Data API」 →
AI + データ + セキュリティ。）

### Step A.3：集計

ジャンル別エントリ数を数える。パーセンテージを計算。

アウトプット：

```
ジャンル分布（Matrix B 集計）：

| ジャンル | エントリ | % |
|---|---|---|
| AI 関連 | X | Y% |
| アーキテクチャ刷新 | X | Y% |
| マルチプロダクト | X | Y% |
| データ基盤 | X | Y% |
| ... | ... | ... |
```

### Step A.4：期待現実と比較

バランスのとれた分布（経験則、領域による）：

- AI：20-40%（高ければ AI フォーカスカンファレンス；低ければ広範囲）
- アーキテクチャ / モダナイゼーション：15-25%
- マルチプロダクト：10-20%
- データ：5-15%
- 信頼性：5-10%
- セキュリティ：5-10%
- インフラ：5-15%
- 組織：10-15%

いずれかのジャンルが >50% なら、それはバイアスの強いシグナル（その領域を再リサーチ、
または不足ジャンルを意識的に偵察）。

### Step A.5：Phase 4.6 の抜け論点チェックへ供給

Phase 4.6（Step 4.6.7）は、構築中のテーマ構造に対する「期待現実」として
この分布を使う。テーマ構造は概ねこの分布を反映すべき。

## 使うテンプレート

## 検証

- [ ] タグ付けは M:N（1 エントリが複数ジャンルを持てる）
- [ ] エントリ総数が Matrix B の実際の「知りたいこと」数と一致
- [ ] 期待分布との比較メモが少なくとも 1 つ

## 注意すべき失敗パターン

- 単一ジャンルタグ付け（エントリが実際は複数ジャンルにまたがるのに）（シグナル損失）
- 客観的な外部代理ではなく「チームが望むもの」と比較
