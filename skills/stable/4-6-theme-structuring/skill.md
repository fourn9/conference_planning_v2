# スキル：Phase 4.6 — セッションテーマ構造化

## 目的

クロスカットテーマ（Phase 4）を、CFP 応募者と登壇者が自分を位置付けられる
階層構造（大 → 中 → 小テーマ）に変換する。

## このスキルを呼び出すとき

- Phase 4 と Phase 4.5 が `completed`
- `conference-planning.yml` に `phase_4_6_session_theme_structuring: pending`

## 手順（エージェント向け）

### Step 4.6.1：入力レビューと焦点設定

収集：

- コアテーマ（Phase 4）
- audience-shape クラスタ（Phase 4-c）
- ターゲット聴衆の成長軸（増やしたい職種）
- 過去セッション分布（あれば）
- 内部検証の洞察（Phase 4.5）

過去データなしなら：
- `skills/analysis/genre-aggregation/` を使用
- オプションで `skills/analysis/adjacent-conferences/`

その上で質問：**「カンファレンスは一文で何について？」** これがキャッチコピー候補。

### Step 4.6.2：大テーマ仮設定（3〜5）

デフォルト分割は Why/What、How、Who、その他：

- **戦略・設計思想**（Why/What）
- **実装・運用**（How）
- **組織・人**（Who）
- **その他 / 補足**（時代を超えた原則、海外動向 等）

3 + その他で 4 つが最も安定。5 つ以上は CFP 応募時に混乱を生む。

### Step 4.6.3：粒度の非対称性を許容

大テーマ間で均等なカウントを強制しない：

- 戦略 ≈ 3 中テーマ（抽象で OK）
- 実装 ≈ 5〜6 中テーマ（専門領域）
- 組織 ≈ 4 中テーマ（階層：マクロ → マイクロ → 個人）
- その他 ≈ 1〜2 中テーマ

**重要**：この非対称性はフレームワークの最も強い意見。カウントを「整える」誘惑に抵抗する。

### Step 4.6.4：中テーマ軸選び（複数試行）

各大テーマについて、軸スタイルを選ぶ：

| 軸 | 例 | 適する |
|---|---|---|
| 時間 | 未来 / 現在 / 過去 + 未来 | 戦略 |
| レイヤー | アプリ / データ / 運用 / インフラ / セキュリティ | 実装 |
| 判断 | 拡張 / 統合 / 分離 | マルチプロダクト戦略 |
| 対立 | 自律 vs 統制 / 同期 vs 非同期 | 哲学テーマ |
| フェーズ | 立ち上げ / スケール / 成熟 | 事業段階テーマ |
| 主体 | 人 / 決定論的 / AI | 責任テーマ |
| 階層 | マクロ / 中 / マイクロ / 個人 | 組織テーマ |

**大テーマあたり 3〜5 候補軸を試行**、その後ターゲット聴衆のメンタルモデルに
合うものを選ぶ。

### Step 4.6.5：小テーマ = 「議論ポイント」

小テーマは **実行時に実際に議論されること** であるべき、ラベルではない：

| ✅ OK | ❌ NG |
|---|---|
| サービス境界の引き直しとモジュラモノリスへの揺り戻し | 「マイクロサービス」（ラベルだけ） |
| 共有 DB からのデータ移行、停止なし | 「レガシー移行」（曖昧すぎ） |
| SLO/SLI 設定と組織浸透 | 「SRE」（ラベル） |
| 「動かしながら変える」 — 停止なしモダナイゼーション | 「モダナイゼーション」（ラベル） |

各中テーマ：4〜5 小テーマ。

### Step 4.6.6：批判的見直しループ（最低 ×2）

構造全体について、問う：

| チェック | 質問 |
|---|---|
| 粒度 | 大／中／小のギャップは軸ごとに適切か？ |
| MECE 性 | 重複？ ギャップ？ |
| 固有名詞 | 「[企業]-style X」が漏れていないか？ |
| 時代性 | ターゲットイベント日の現実を反映しているか？ |
| ターゲット整合 | これらはターゲット職種の実際の議論か？ |
| 二項対立トラップ | 3 つやスペクトラムの方が誠実なときに 2 つに強制していないか？ |
| 議論 vs ラベル | 小テーマは議論であって、カテゴリではないか？ |
| AI バイアス | AI が適合しないテーマにまで AI を強制していないか？ |

最低 2 パス。バイアスはしばしば 2 回目の見直しでのみ現れる。

### Step 4.6.7：抜け論点チェック

クロスリファレンス：

- 過去セッション分布（`use_historical_data: true` なら）
- 隣接カンファレンス分布（スキル：`skills/analysis/adjacent-conferences/`）
- Matrix B ジャンル分布（スキル：`skills/analysis/genre-aggregation/`）
- ヒートマップトップテーマ（基調候補テーマがどこかに登場するか？）
- ターゲット聴衆の成長軸（増やしたい職種 — 関心がカバーされているか？）

実質的なものが欠けていれば追加または並べ替え。

### Step 4.6.8：横軸 + キャッチコピー

- **横軸**：クロスカットの設計思想（モジュラモノリス vs マイクロサービス選択論、
  自律 vs 統制 等）。大テーマの隣に、すべての大テーマの「容器」として位置する。
- **キャッチコピー**：構造全体の一文サマリ。カンファレンスの「なぜ」を内包すべき。

### Step 4.6.9：アウトプット

ユーザーが作業ディレクトリに保存：

```
[あなたの作業ディレクトリ]/session-theme-structure.md
```

`templates/session-theme-structure.md` 形式で。

## 注意すべき失敗パターン

- **均等粒度を強制**（#16）：ここで最も多い失敗
- **1 テーマ = 1 セッション思考**（#17）：テーマはソート軸、セッション割り当てではない
- **AI 過剰表現**（#18）：小テーマの 70%+ で AI が出てきたらドリフトしている
- **二項対立のフレーミング**（#19）：「X vs Y」が時々第 3 の選択肢を隠す
- **ラベルだけの小テーマ**（#20）：「データ基盤」は小テーマではない；
  「マルチプロダクト間のデータ基盤ジャンル統一」は小テーマ
