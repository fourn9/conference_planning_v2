# ADR-0007: ディープリサーチを外部 LLM 貼り付けではなく Claude Code 内ワークフローで完結させる

- **Status**: Accepted
- **Date**: 2026-06
- **Tags**: research, tooling, workflow
- **Related**: [ADR-0004](0004-matrix-b-research-hardgate.md), [ADR-0006](0006-research-knowledge-pipeline.md)

## Context（背景・問題）

ADR-0006 は「クラウド LLM（claude.ai 等）で Deep Research を実行し、共有 Vault で Claude Code と接続する」
パイプラインを定めた。Vault による接続は機能するが、**実行そのものが外部 LLM 依存**であることに起因する
運用課題が残っていた：

1. **手動貼り付けが必須** — プロンプト（数百行）を人間が外部 LLM にコピペし、完了を待って回収する
2. **書き込み経路が壊れやすい** — クラウド側からのストレージ書き込み（MCP 経由）でファイル破損・配置ゆれが発生した実績がある
3. **実行過程が記録に残らない** — 何件探索し、何件を検証し、何を検証せず打ち切ったかが成果物から読めない
4. **採否ハードゲート（ADR-0004）の検証が1セッションの自己申告**になり、候補単位の独立検証ができない

一方、Claude Code のマルチエージェントワークフロー（探索エージェントの fan-out → 候補ごとの独立検証
エージェント）で登壇候補リサーチ（3テーマ・探索12角度）を実行したところ、42 サブエージェント・約36分で
確定27名（候補単位のゲート検証・出典 URL 実在確認つき）を出力でき、外部 LLM 実行と同等以上の品質を
プロンプト貼り付けなしで達成できることを実証した。

## Decision（決定）

1. ディープリサーチ（登壇候補発掘等）は **Claude Code 内のマルチエージェントワークフローで完結**させる。
   外部 LLM（claude.ai 等）への貼り付け実行はフォールバック（Claude Code で実行できない場合）に限る。
2. 実行形は `.claude/workflows/speaker-deep-research.js`。探索角度・テーマ別ゲート判別ルール・既出除外名簿・
   検証上限を `args` で注入する（リサーチ内容をスクリプトにハードコードしない）。
3. リサーチ仕様（プロンプト）はこれまで通り `design-prompt` スキルで設計し、仕様書として保存する。
   プロンプト＝仕様書の規約（ADR-0006）は維持し、ワークフローはそれを実行系として読む。
4. 結果は Vault へ**ローカル同期パス経由で直接書き込む**（クラウド側 MCP 書き込みを使わない。
   ローカル → クラウドはストレージクライアントの自動同期に任せる）。
5. 結果ファイル冒頭に**実行メモ**を必記する：実行系・検証範囲の上限（cap）・実施できなかった突合
   （例：スポンサー予定企業リスト未提供）・既出フラグの説明。

## Consequences（結果・トレードオフ）

- ✅ プロンプト貼り付け往復が消え、設計 → 実行 → 整形 → 格納が1セッションで完結する
- ✅ 候補ごとに独立エージェントがゲート検証するため、自己申告でない検証になる。実行過程（探索件数・検証件数・ドロップ分）が記録に残る
- ✅ クラウド側書き込み起因のファイル破損経路を回避できる
- ⚠️ 検証上限（cap）を超えた探索ヒットは未検証リードになる — 結果ファイルに必ず記録し、必要なら追検証を回す
- ⚠️ 既出（過去リサーチ採用済み）の除外は **探索エージェントのプロンプトに名簿として渡す**（下流の突合だけだと重複発掘で検証コストを浪費する）
- レビュー・改善ループ（`review-output` → `iterate-from-failures`）は実行系によらず共通に適用する
