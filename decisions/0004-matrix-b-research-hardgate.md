# ADR-0004: Matrix B リサーチのハードゲートとプロセス規律

- **Status**: Accepted
- **Date**: 2026-05
- **Tags**: research, methodology, validation
- **Extends**: [ADR-0002](0002-research-source-evaluation.md), [ADR-0003](0003-3d-research-framework.md)

## Context（背景・問題）

Matrix B 初期実装では、生リサーチからの候補採用が約 15-20% にとどまった。
失敗モードは 4 パターンに分かれる：

1. **手法名キーワード検索** で概念記事、ハンドブック、参加者まとめが
   引っかかった — 当事者著者の作品ではない
2. **隣接職種のリーク** で VPoE ／アーキテクト／データ／ ML ／スタートアップ CTO が
   BE/FS や TL 行にクロスインした
3. **会社レベル（プロダクトレベルではない）のフェーズ判断** が事業ユニット固有の
   作業を誤分類した
4. **業界トレンド／他社まとめ記事** が当事者判断として数えられた

明示的ゲートなしには、フレームワークが再利用されるたびにこれらの失敗モードが再発する。

## Decision（決定）

すべての Matrix B 候補 URL ／ソースに 7 つの **ハードゲート**（G1-G7）を適用：

| ゲート | チェック |
|---|---|
| G1 | 著者の職種が識別可能（セルの職種カテゴリに一致） |
| G2 | アウトプットが 2024-01 以降（スタートアップ：2021+） |
| G3 | コンテンツが自社プロダクトでの **当事者** 経験を反映 |
| G4 | URL が検証可能、著者が実在、引用が再現できる |
| G5 | **厳密** 職種カテゴリ適合（隣接職種はこのセルでは不可） |
| G6 | タイトルを超えたコンテンツ実質（本物のエンジニアリング主張） |
| G7 | 目的との文脈一致（キーワードだけではない） |

加えて 6 つの **プロセス規律**：

- (a) 検索 → 本文読み → 3 点メモ → 採否判定。スニペットで採用しない。
- (b) Matrix A は **選定基準の材料**、closed list ではない。Matrix A の推論を引用しつつ、
  その企業を超えて検索する。
- (c) 1 記事 = 複数「知りたいこと」エントリ（M:N）可。エントリごとに引用。
- (d) **「該当者なし」許容**。空セルは Matrix A 仮説の再チェック候補になる。
- (e) **厳格な実在確認**：各 URL にアクセス、アクセス日を記録。
- (f) **選定基準 = 3 要素**：サービス属性／プレス／アーキシグナル。
  含めない：組織条件／アーキ前提決め打ち／セル内容／経営宣言。

加えて **隣接職種ルール**：

| 隣接役割 | デフォルト配置 |
|---|---|
| VPoE | EM 行、または独自セクション |
| アーキテクト | TL 行（職務記述が重なる場合のみ） |
| データエンジニア／データプラットフォームエンジニア | データ行 |
| ML エンジニア | 前提知識セクション（Matrix B 行ではない） |
| TL ハンズオン付き PMF 前 CTO | TL 行 OK |
| IT 大手 / 成長期 CTO | Matrix A のみ（Matrix B 行ではない） |

加えて **前提知識セクション**（Matrix B 行と並列）：カテゴリ外の記事（ML、
プラットフォームエンジニアリング 等）を目的別に整理、7 職種行ではない。

## Consequences（結果・影響）

### Positive
- 生リサーチの採用率が劇的に改善（15-20% → ゲート強制後はずっと高い）
- ペルソナ作業に高忠実度のソース
- 行間不整合（同じ著者が TL では OK、BE/FS では NG）が体系的に捕捉される
- 「該当者なし」トラッキングが Matrix A の弱いセルを再リサーチ用に浮上させる

### Negative / Trade-offs
- URL ごとの評価コスト増加
- 却下候補が蓄積する — `rejected-candidates.md` トラッキング必要
- エッジケース（「TL コンテンツのように読める VPoE 記事」）が明示的ルール参照を要する

### Risks
- 厳密 G2（2024+）が依然として現在標準である 2023 ソースを却下するかも
  （緩和プロセスは文書化、実際にはまれ）
- G5 厳格性は記事が明確に関連していても職種カテゴリが 1 行ずれていると懲罰的に感じる

## Alternatives Considered（検討した代替案）

### 代替案 A：緩いゲート、20-30% ゲートバイパス率を許容

却下 — 目的に反する；循環推論が再主張される。

### 代替案 B：単一ゲートプロセス（「当事者か？」だけ）

却下 — 当事者性だけでは、キーワードマッチ、隣接職種、会社レベル混同を捕捉できない。

### 代替案 C：フリー検索後のゲートベースフィルタリング

代替案 A と同じ — ゲーティングが遅すぎる時点で発生し、採用候補の多くがすでに弱い。
